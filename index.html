Human Computer Interaction Project #1 Report

 

Picture of the Roper microwave in apartment: 

 

Issues with current design:

  • Defrost functionality: defrost works by pushing the defrost button and either using the number pad to select the type of food you are defrosting (meat, poultry, or fish) or pushing the defrost button once again for meat, twice for poultry, and three times for fish. The user is prompted the enter how many lbs. of the food item they wish to defrost. The way it is setup now, it is assuming that the user knows what numbers on the number pad are for what food item. It also assumes that the user can just push the defrost button again to cycle through all three of the options. The last issue with this functionality is that the LED does not give a clear message “Enter lbs.”, it simply says “Enter” followed by -.- with quantity and lbs. underneath which is a bit confusing. 

 

  • Reheat functionality: reheat works in a similar fashion to the defrost function but is a bit more confusing because the reheat function gives the user 5 options of foods to reheat. It works the same way as the defrost regarding how the user can select the foods to reheat (number pad or by pushing the reheat button again and again to cycle through the food options). A big problem with this is the abbreviations the microwave uses to represent a food option. CSRLE is supposed to stand for casserole I believe, I could be wrong though. I do not like that the microwave uses confusing abbreviations. The reheat function is a little bit better regarding how the microwave prompts the user to input what amount they are wanting to reheat. For example, for pizza the LED says Enter SLICES - , which is straight forward. 

 

  • Cook time/power functionality: cook power works by pushing the cook power button after the user has input a time using the cook time button. The user can use the number pad to enter 0-10 in for the cook power or they can push the cook power button again and again to cycle down from 10 to 0 all the way back to 10 again. The user can also push the cook power button without having a time entered, cycle through the cook power options, and then push the cook time button to enter the time. I think this is bad because why have two different ways to do the same outcome (cooking food)?

 

  • LED display: The LED display is way too small for the information it needs to output to the user. The display text is super small making it difficult to read/use. It is obvious why this is a bad thing.

 

  • Popcorn button: I honestly do not see the need for this button at all. Generally, popcorn has instructions on how long to cook the popcorn. Also, the microwave gives the user the option between 3.50, 3, 1.75 when cycling through the popcorn options which do not know what they represent, maybe ounces? They do not represent time amounts such as minutes: seconds.  

 

Common/rare uses of a microwave:

  • Cooking food

 

  • Reheating food

 

  • Defrosting food

 

  • An overhead oven light

 

  • A clock

 

  • An overhead stovetop smoke fan 

            I feel like the interface makes the use of a microwave as an overhead oven light, stovetop smoke fan, and clock relatively easy with the way the functionality of the microwave buttons work. I think the interface makes the use of a microwave to cook, reheat and defrost food a little difficult for the       reasons I have explained above in my “Issues with current design” section.

 

 Common sequence of actions by user:

  • To cook food: open microwave door, place food inside, shut door, push cook time button, enter a cook time, (rarely change the cook power by pushing cook power button and entering a cook power from 0-10), push start button.

 

  • To defrost food: open microwave door, place food inside, shut door, push defrost button, make a food selection, enter in a lbs. amount, pushing start button.

 

  • To reheat food: same as defrost food sequence of actions except using the reheat button. 

Microwave user friendliness/feedback:

  • The microwave prompts user to use number pad after pushing the defrost or reheat button by outputting PAD on the LED display.

 

  • The microwave prompts the user to enter a number of lbs./cups/slices of food after pushing defrost or reheat button, making a food selection by using the LED display and outputting words/symbols like Enter CUPS -, Enter -.- quantity/lbs., etc. 

 

  • The microwave shows the user the current cooking power when they have pushed either cook time button or cook power button.

 

  • The microwave will beep once the time input by the user has ran out and the food is ready to be removed.

 

Common mistakes with current design:

  • I often select the wrong food item when using the defrost/reheat functions or I avoid using them altogether and just use the regular cook and change the power based on what I need the microwave to do.

 

Suggestions to improve microwave control interface:

  • Completely remove the popcorn button, I have never used it, the options the microwave gives the user for the function is vague. I do not see the need for it on the interface.

 

  • Increase the size of the LED display screen. It is hard to read the very little instructions/feedback the microwave gives the user as is. If the LED display were larger, the display could show things like function options so it would be more user friendly. An example of this would be for the defrost or reheat functions. If the LED display were larger, the microwave could show something like: 

 

“SELECT DEFROST OPTION ON NUM PAD: 

                        MEAT:         1

                        POULTRY:  2

                        FISH:            3”

 

  • Remove the cook power button and have a up and down button labeled “Adjust Cook Power” so that there is one way to start a regular microwave cook session: Push Cook Time, enter time, push up or down to adjust cook power if the user wants to, and push the start button to start microwave.

 

  • Defrost and reheat buttons can no longer be used to cycle through the preset selection options. These buttons will be used to output the correct presets for either function (meat, poultry, and fish for defrost and soup, casserole, baked goods, etc. for reheat), which the user will select by pushing the appropriate number on the number pad. Depending on whether the defrost or reheat button was pushed, once a number has been pushed the LED display will prompt the user to enter the corresponding amount for selected item. For example, if poultry were chosen in the defrost function, the LED would show something like:

 

“ENTER AMOUNT TO DEFROST:

                        _._ lbs.”
